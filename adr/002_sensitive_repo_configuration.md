# 002: リポジトリ固有情報をハードコードしない方針

- 日付: 2025-11-08
- ステータス: Accepted

## コンテキスト
- 本プロジェクトは OSS として公開されるため、ユーザー固有の GitHub リポジトリ名や承認ルールなどをソースコードに直接含めると情報漏えいにつながる。
- `GITHUB_TARGET_REPOS` や承認しきい値など、利用者ごとに異なる設定が増えており、コード内に既定値を埋め込むとメンテナンス負荷や誤用のリスクが高まる。

## 決定
- 特定の GitHub リポジトリ名、承認しきい値など機密性のある設定値は、ソースコードにハードコードしない。
- これらの設定は環境変数（`.env` など）またはリポジトリ未コミットの設定ファイル（例: `GITHUB_APPROVAL_THRESHOLDS_FILE` で指定）から読み込む。
- OSS として共有されるテンプレート類（`.env.template`, ADR, README など）には、具体的なリポジトリ名ではなくプレースホルダー例のみを記載する。

## 影響
- 今後、設定値が必要な機能を追加する際は、必ず環境変数またはコミット対象外ファイルから読み込む仕組みを先に用意する。
- 既存コードに特定リポジトリ名が残っている場合は、環境設定へ移行し、ここで定義した方針に従って削除・置き換えを行う。
