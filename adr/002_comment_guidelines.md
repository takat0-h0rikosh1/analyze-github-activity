# 002: コメント記載方針

- 日付: 2025-02-14
- ステータス: Accepted

## コンテキスト
- 本プロジェクトでは R スクリプトを複数の開発メンバーが共同で保守する。
- コードを参照するメンバーの中には英語への習熟度が高くない者も存在する。
- コメントが英語中心になると仕様理解やレビューに時間を要する可能性がある。

## 決定
- 原則としてコメントは日本語で記載する。
- コメントの目的は、言語リテラシーが高くないメンバーでも挙動や背景を理解できるように丁寧に説明することとする。
- 必要に応じ簡潔な英語併記を許容するが、日本語説明を優先する。

## 影響
- 新規に追加するコメントは日本語説明を基本とする。
- 既存コメントが英語のみの場合でも、今後修正時には日本語説明を追加することを検討する。
- 外部ライブラリの名前や API 固有の表現は英語のままで差し支えない。
